React Routing

React Router v6 — nested routes, protected routes, lazy loading, and navigation hooks

React Router v6 Setup

React Router v6 is the standard client-side routing solution. It uses JSX-based route definitions and a hooks-first API.

v6 key changes: Switch replaced by Routes, component prop replaced by element, nested routes use Outlet, and useHistory is now useNavigate. 
import { BrowserRouter, Routes, Route, Link, NavLink, Outlet } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <nav>
        <NavLink to="/" className={({isActive}) => isActive ? 'active' : ''}>Home</NavLink>
        <NavLink to="/products">Products</NavLink>
        <NavLink to="/about">About</NavLink>
      </nav>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/products" element={<Products />} />
        <Route path="/products/:id" element={<ProductDetail />} />
        <Route path="/about" element={<About />} />
        <Route path="*" element={<NotFound />} />  // catch-all 404
      </Routes>
    </BrowserRouter>
  );
}

Nested Routing with Outlet

Nested routes allow child routes to render inside a parent layout component via <Outlet /> — perfect for dashboards, admin panels, and multi-section layouts.

// Dashboard with nested routes
<Route path="/dashboard" element={<DashboardLayout />}>
  <Route index element={<DashboardHome />} />      // /dashboard
  <Route path="profile" element={<Profile />} />   // /dashboard/profile
  <Route path="settings" element={<Settings />} /> // /dashboard/settings
</Route>

function DashboardLayout() {
  return (
    <div style={{ display: 'flex' }}>
      <Sidebar />
      <main>
        <Outlet /> // nested routes render here
      </main>
    </div>
  );
}
index route: The index prop marks the default child route — rendered when the parent path matches exactly (no additional segment).

Dynamic Routes + useParams

URL parameters (prefixed with :) are accessible via useParams() inside any component rendered by that route.

function ProductDetail() {
  const { id } = useParams();
  const { data, loading } = useFetch(`/api/products/${id}`);

  if (loading) return <Spinner />;
  return (
    <div>
      <h1>{data.name}</h1>
      <p>{data.description}</p>
    </div>
  );
}

Protected Routes

Protected routes guard pages that require authentication. They redirect unauthenticated users to login while preserving the intended destination for post-login redirect.

function ProtectedRoute({ children }) {
  const { user } = useAuth();
  const location = useLocation();

  if (!user) {
    return <Navigate to="/login" state={{ from: location }} replace />;
  }

  return children;
}

// Usage — wrap any route that needs auth
<Route path="/dashboard" element={
  <ProtectedRoute>
    <Dashboard />
  </ProtectedRoute>
} />

// After login — redirect back to the page the user tried to visit
function LoginPage() {
  const navigate = useNavigate();
  const location = useLocation();
  const from = location.state?.from?.pathname || '/';

  const handleLogin = async () => {
    await login();
    navigate(from, { replace: true });
  };
}

Lazy Loading Routes

Route-based code splitting is the highest-impact performance optimisation for large SPAs — each route becomes a separate JS chunk only loaded when visited.

import { lazy, Suspense } from 'react';

const Home = lazy(() => import('./pages/Home'));
const Dashboard = lazy(() => import('./pages/Dashboard'));
const Analytics = lazy(() => import('./pages/Analytics'));

<Suspense fallback={<PageLoader />}>
  <Routes>
    <Route path="/" element={<Home />} />
    <Route path="/dashboard" element={<Dashboard />} />
    <Route path="/analytics" element={<Analytics />} />
  </Routes>
</Suspense>
Tip: Place <Suspense> as high as possible (wrapping all routes) so a single fallback handles all lazy route loading.

useNavigate + useSearchParams

These hooks handle programmatic navigation and URL query string management — essential for search pages, filters, and pagination.

function SearchPage() {
  const navigate = useNavigate();
  const [searchParams, setSearchParams] = useSearchParams();
  const query = searchParams.get('q') || '';
  const page = Number(searchParams.get('page')) || 1;

  return (
    <div>
      <input
        value={query}
        onChange={e => setSearchParams({ q: e.target.value, page: '1' })}
      />
      <button onClick={() => navigate(-1)}>Back</button>
      <button onClick={() => setSearchParams({ q: query, page: String(page + 1) })}>
        Next Page
      </button>
    </div>
  );
}
useNavigate(-1) — go back (like browser back button)
useNavigate(1) — go forward
navigate('/path', { replace: true }) — replace history entry (no back)
useSearchParams keeps filters in the URL, making them shareable and bookmarkable without any extra state management.

Interview Questions

What is the difference between BrowserRouter and HashRouter?
BrowserRouter uses the HTML5 History API (pushState) for clean URLs like /products/1 — requires server-side fallback config. HashRouter uses # fragments (/#/products/1) — works without server config but looks less clean and breaks anchor links.
How does Outlet work in React Router v6?
<Outlet /> is a placeholder in a parent route's component where matching child routes render. It's how nested routing is implemented — the parent provides shared layout, and the outlet swaps in the active child content.
How do you redirect after login while preserving the intended URL?
In the ProtectedRoute, pass state={{ from: location }} to the Navigate component. After successful login, read location.state?.from?.pathname and call navigate(from, { replace: true }) to send the user to their original destination.
What is the purpose of the index route?
An index route renders when its parent route path matches exactly with no additional segments. It's the default child — equivalent to a route with path="" — and avoids needing to repeat the parent path.
Why should you use NavLink instead of Link for navigation menus?
NavLink automatically receives an isActive boolean in its className and style callbacks, letting you apply active styles without manually comparing the current URL. Link provides no active state awareness.
How does code splitting with lazy() reduce initial load time?
lazy() wraps a dynamic import(), telling the bundler (Webpack/Vite) to emit each lazy-loaded module as a separate chunk. On initial page load, only the chunks for the visited route are downloaded — other route bundles load on demand when navigated to.
What does replace: true do in useNavigate?
navigate('/path', { replace: true }) replaces the current entry in the browser history stack instead of pushing a new one. This means the back button won't return to the replaced page — useful for login redirects and wizard flows where back-navigation would be confusing.
How do you persist filters/pagination in the URL with React Router?
Use useSearchParams to read and write URL query parameters (?q=react&page=2). Call setSearchParams({ q: value }) to update — it pushes a new history entry preserving the URL, making the state shareable, bookmarkable, and recoverable on refresh.
