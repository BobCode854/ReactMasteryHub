JWT Authentication Flow

JWT (JSON Web Token) is a compact, self-contained token that carries user identity information. It's signed by the server so the client can trust its contents.

1
User Fills Login Form
Email + password entered in the React form component
2
POST /api/login with Credentials
Client sends { email, password } to backend API
3
Server Validates → Returns JWT Token
Server checks credentials, creates signed token with user data
4
Client Stores Token
localStorage.setItem('token', data.token) — or httpOnly cookie (more secure)
5
Every API Request Includes Token
Authorization: Bearer <token> header on all requests
6
Server Validates Token → Returns Data
Server verifies signature, checks expiry, returns protected data
JWT Structure: A JWT has 3 parts separated by dots: header.payload.signature. The header says which algorithm was used, the payload contains user claims (id, role, expiry), and the signature proves it wasn't tampered with.
Complete JWT Auth Implementation

Build a full AuthContext that handles login, logout, and persisting auth state across page refreshes.

const AuthContext = createContext(); export function AuthProvider({ children }) { const [user, setUser] = useState(null); const [loading, setLoading] = useState(true); const navigate = useNavigate(); // Check token on app load useEffect(() => { const token = localStorage.getItem('token'); if (token) { api.get('/auth/me', { headers: { Authorization: `Bearer ${token}` } }) .then(data => setUser(data.user)) .catch(() => localStorage.removeItem('token')) .finally(() => setLoading(false)); } else { setLoading(false); } }, []); const login = async ({ email, password }) => { const data = await api.post('/auth/login', { email, password }); localStorage.setItem('token', data.token); setUser(data.user); navigate('/dashboard'); }; const logout = () => { localStorage.removeItem('token'); setUser(null); navigate('/login'); }; return ( <AuthContext.Provider value={{ user, login, logout, loading }}> {loading ? <FullPageSpinner /> : children} </AuthContext.Provider> ); } // Custom hook for consuming auth context export function useAuth() { const context = useContext(AuthContext); if (!context) throw new Error('useAuth must be used within AuthProvider'); return context; }
Axios Interceptor for Auto Token Injection
// src/api/axios.js — attach token to every request automatically import axios from 'axios'; const api = axios.create({ baseURL: process.env.REACT_APP_API_URL }); api.interceptors.request.use((config) => { const token = localStorage.getItem('token'); if (token) config.headers.Authorization = `Bearer ${token}`; return config; }); // Auto-logout on 401 Unauthorized api.interceptors.response.use( (response) => response.data, (error) => { if (error.response?.status === 401) { localStorage.removeItem('token'); window.location.href = '/login'; } return Promise.reject(error); } ); export default api;
Protected Routes

Protected routes check if the user is authenticated (and optionally if they have the required role) before allowing access. Unauthorized users are redirected to login.

function ProtectedRoute({ children, roles }) { const { user } = useAuth(); const location = useLocation(); if (!user) { // Save where they were going — redirect back after login return <Navigate to="/login" state={{ from: location }} replace />; } if (roles && !roles.includes(user.role)) { return <Navigate to="/403" replace />; } return children; } // Usage in router <Routes> <Route path="/login" element={<Login />} /> <Route path="/dashboard" element={<ProtectedRoute><Dashboard /></ProtectedRoute>} /> <Route path="/admin" element={ <ProtectedRoute roles={['admin']}> <AdminPanel /> </ProtectedRoute> } /> </Routes>

OAuth with Google

OAuth allows users to authenticate with their existing Google (or GitHub, Facebook, etc.) account without creating a new password. The Google token is sent to your backend for verification.

import { GoogleOAuthProvider, GoogleLogin } from '@react-oauth/google'; function App() { return ( <GoogleOAuthProvider clientId={process.env.REACT_APP_GOOGLE_CLIENT_ID}> <LoginPage /> </GoogleOAuthProvider> ); } function LoginPage() { const { login } = useAuth(); return ( <GoogleLogin onSuccess={async (credentialResponse) => { // Send Google token to your backend for verification await login({ googleToken: credentialResponse.credential }); }} onError={() => console.log('Google Login Failed')} /> ); } // Backend verifies the Google token and issues your own JWT // POST /api/auth/google — body: { googleToken } // Server: verify with Google → create/find user → return your JWT
OAuth Flow Summary: Your app → Google → User grants permission → Google sends token to your app → Your app sends token to YOUR backend → Backend verifies with Google → Backend creates session/JWT → User is logged in.
Role-Based Access Control (RBAC)

RBAC restricts what users can see and do based on their assigned role. Implement it at the route level (ProtectedRoute) AND at the UI level (hide/show elements).

// Permission-based component rendering function Can({ perform, children, fallback = null }) { const { user } = useAuth(); const permissions = { admin: ['read:users', 'write:users', 'delete:users', 'read:reports'], manager: ['read:users', 'write:users', 'read:reports'], user: ['read:users'], }; const userPermissions = permissions[user?.role] || []; if (!userPermissions.includes(perform)) return fallback; return children; } // Usage — clean declarative permission checking <Can perform="delete:users" fallback={<span>No permission</span>}> <DeleteButton onClick={handleDelete} /> </Can> <Can perform="read:reports"> <ReportsLink /> </Can>

Token Security
Use httpOnly cookies instead of localStorage — httpOnly cookies can't be accessed by JavaScript, protecting against XSS attacks. localStorage is accessible to any script on the page.
Set short token expiry + use refresh tokens — Access tokens should expire in 15-60 minutes. Refresh tokens (stored in httpOnly cookies) silently obtain new access tokens without requiring re-login.
Always validate on the server — Never trust client-side role checks alone. Every API request must validate the JWT and check permissions server-side. Client-side checks are only for UX (hiding/showing UI).
Implement token revocation — JWTs are stateless, so you can't "invalidate" them. Use short expiry + a blocklist for compromised tokens, or switch to opaque tokens with server-side sessions for high-security apps.
Common Mistakes
Storing sensitive data in JWT payload — The payload is base64 encoded, NOT encrypted. Anyone can decode it. Never put passwords, credit cards, or secrets in the token.
Only checking auth on the frontend — A user can bypass React's ProtectedRoute by calling your API directly. Backend must always validate permissions independently.
Not handling token expiry gracefully — When a token expires, show the user a "session expired" message and redirect to login, rather than silently failing or showing cryptic errors.
Exposing auth state in URL — Never put tokens in URL query parameters — they appear in browser history, server logs, and Referer headers.
