Calculator (iOS Style)

Beginner useState event handling

An iOS-style dark calculator with circular buttons and orange operator highlights. Manages prev, current, operator, and fresh-entry state flags for correct chained calculations.

Key Learning: Multiple useState variables work together as a mini state-machine. The fresh flag tells the app whether the next digit press should start a new number or append to the current display — this pattern is essential for calculators and text editors.

String(x / y) : 'Error'; default: return b; } }; const handleBtn = (btn) => { if (btn === 'AC') { setDisplay('0'); setPrev(null); setOp(null); setFresh(true); return; } if (btn === '+/-') { setDisplay(d => String(parseFloat(d) * -1)); return; } if (btn === '%') { setDisplay(d => String(parseFloat(d) / 100)); return; } if (['+','−','×','÷'].includes(btn)) { setPrev(display); setOp(btn); setFresh(true); return; } if (btn === '=') { if (prev !== null && op) { const result = calculate(prev, display, op); setDisplay(result); setPrev(null); setOp(null); setFresh(true); } return; } if (btn === '.') { if (fresh) { setDisplay('0.'); setFresh(false); return; } if (!display.includes('.')) setDisplay(d => d + '.'); return; } // digit if (fresh) { setDisplay(btn); setFresh(false); } else setDisplay(d => d === '0' ? btn : d.length < 12 ? d + btn : d); }; const isOp = (b) => ['+','−','×','÷'].includes(b); const isTop = (b) => ['AC','+/-','%'].includes(b); const btnStyle = (btn) => ({ width: btn === '0' ? '1fr 1fr 1fr' : '1fr 1fr 1fr 1fr', gap: 12, marginBottom: 12 }}> {row.map(btn => ( <button key={btn} style={btnStyle(btn)} onClick={() => handleBtn(btn)} onMouseDown={e => e.currentTarget.style.opacity = '.7'} onMouseUp={e => e.currentTarget.style.opacity = '1'}> {btn} </button> ))} </div> ))} </div> ); } export default Calculator;

Snake Game

Beginner useState useEffect useRef game loop

Classic Snake on a 20×20 grid. Arrow keys control direction, score increases with food, speed ramps up every 5 points, and collision with walls or self ends the game.

Key Learning: useRef for game state (direction, snake body, food) avoids stale closure issues inside setInterval callbacks. React state (useState) is only used for values that should trigger a visual re-render (score, game-over screen, the rendered grid).
