          <p><code>BrowserRouter</code> uses the HTML5 History API (<code>pushState</code>, <code>replaceState</code>, <code>popstate</code> event) to create clean URLs like <code>/users/profile</code>. <code>HashRouter</code> uses the URL hash fragment (the <code>#</code> part) to create URLs like <code>/#/users/profile</code>. BrowserRouter produces cleaner, more professional-looking URLs and is the standard choice for modern web apps. HashRouter is useful when you cannot configure the server to serve the app from arbitrary paths — since the hash is never sent to the server, all hash-based routes work without server configuration. For production apps deployed to a server that can be configured for SPA routing (redirecting all paths to index.html), always prefer BrowserRouter.</p>

          <p>A <code>Route</code> is a React Router component that renders its element when its <code>path</code> matches the current URL. In React Router v6, routes are declared as JSX using <code>&lt;Route path="..." element={&lt;Component /&gt;} /&gt;</code> inside a <code>&lt;Routes&gt;</code> container. The <code>Routes</code> component always renders only the best matching route (unlike v5's <code>Switch</code>). Routes can be nested to create hierarchical URL structures. Route paths support dynamic segments (<code>:paramName</code>), optional segments, wildcards (<code>*</code>), and index routes. Routes can also have <code>loader</code> and <code>action</code> functions (data router API, v6.4+) for co-locating data fetching with routing.</p>

          <p>Nested routes allow child routes to render within a parent route's layout, creating hierarchical URL structures where the parent's UI persists while only the child content changes. In React Router v6, you nest routes by placing <code>&lt;Route&gt;</code> elements inside other <code>&lt;Route&gt;</code> elements in your route config. The parent route renders an <code>&lt;Outlet&gt;</code> component that acts as a placeholder where the matching child route's element renders. For example, <code>/dashboard</code> renders <code>DashboardLayout</code> with an <code>&lt;Outlet&gt;</code>, and <code>/dashboard/users</code> renders <code>UserList</code> inside that layout. This eliminates the need for route-level layout HOCs and makes the relationship between routes and layouts explicit in the route config.</p>
<pre class="code-block">// Route configuration with nesting
import { createBrowserRouter, RouterProvider, Outlet } from 'react-router-dom';

function AppLayout() {
  return (
    &lt;div&gt;
      &lt;Header /&gt;
      &lt;Outlet /&gt; {/* child route renders here */}
      &lt;Footer /&gt;
    &lt;/div&gt;
  );
}

const router = createBrowserRouter([
  {
    path: '/',
    element: &lt;AppLayout /&gt;,
    children: [
      { index: true, element: &lt;Home /&gt; },
      {
        path: 'users',
        element: &lt;UsersLayout /&gt;,
        children: [
          { index: true, element: &lt;UserList /&gt; },
          { path: ':userId', element: &lt;UserDetail /&gt; },
        ],
      },
    ],
  },
]);</pre>

        &lt;Outlet context={{ user }} /&gt;
      &lt;/main&gt;
    &lt;/div&gt;

}

function UserSettings() {

          <p>Protected routes redirect unauthenticated users to the login page before allowing access. In React Router v6, the cleanest approach uses a layout route with an authentication check: create a component that checks auth state (from Context, Redux, or a hook) and either renders <code>&lt;Outlet&gt;</code> (authenticated) or <code>&lt;Navigate to="/login" replace&gt;</code> (not authenticated). <pre class="code-block">// Layout route approach
function RequireAuth() {
  const { user, loading } = useAuth();
  const location = useLocation();

  if (loading) return &lt;FullPageSpinner /&gt;;
  if (!user) {
    // Save the intended destination and redirect to login
    return &lt;Navigate to="/login" state={{ from: location }} replace /&gt;;
  }
  return &lt;Outlet /&gt;;
}

// After login, redirect back to intended destination
function Login() {
  const navigate = useNavigate();
  const location = useLocation();
  const from = location.state?.from?.pathname ?? '/dashboard';

  const handleLogin = async (credentials) =&gt; {
    await loginAPI(credentials);
    navigate(from, { replace: true });
  };

          <p>Lazy loading with React Router uses <code>React.lazy()</code> for component lazy loading and <code>Suspense</code> for the loading state. Wrap the lazy-loaded route element in a Suspense boundary — often placed around the <code>&lt;Routes&gt;</code> container. With React Router v6.4's data router API, each route can also have a lazy <code>loader</code> function or the route config itself can use dynamic imports in the <code>lazy</code> property, which loads the component and its loader together when the route is first navigated to. Using route-level code splitting means each page's JavaScript is only loaded when the user navigates to it, significantly reducing the initial bundle.</p>
<pre class="code-block">import { lazy, Suspense } from 'react';
import { createBrowserRouter } from 'react-router-dom';

// Option 1: React.lazy with Suspense
const Dashboard = lazy(() =&gt; import('./pages/Dashboard'));
const Settings = lazy(() =&gt; import('./pages/Settings'));

// Option 2: React Router v6.4 lazy route property (loads component + loader together)
const router = createBrowserRouter([
  {
    path: '/dashboard',
    lazy: async () =&gt; {
      const { DashboardPage, loader } = await import('./pages/Dashboard');
      return { Component: DashboardPage, loader };
    },
  },
]);

// Option 1 usage with Suspense wrapper
&lt;Suspense fallback={&lt;PageSpinner /&gt;}&gt;
  &lt;Routes&gt;
    &lt;Route path="/dashboard" element={&lt;Dashboard /&gt;} /&gt;
    &lt;Route path="/settings" element={&lt;Settings /&gt;} /&gt;

&lt;/Suspense&gt;</pre>

      <div id="routing-q16" class="accordion-collapse collapse" aria-labelledby="heading-routing-q16" data-bs-parent="#acc-routing">
        <div class="accordion-body">
          <p>Route loaders are functions attached to routes via the <code>loader</code> property that fetch data for the route before it renders. When a user navigates to a route, React Router calls the route's loader function first and awaits its result. The component then receives this data via <code>useLoaderData()</code> hook without needing a useEffect-based fetch. This eliminates the "fetch-on-render" waterfall (where a component renders, then fetches, then shows data) by moving data fetching to the navigation phase — React Router starts the fetch as soon as navigation begins, in parallel with loading the JavaScript bundle. Loaders can also return <code>redirect()</code> responses to perform conditional redirects based on data (e.g., redirecting to onboarding if the user has no profile).</p>
<pre class="code-block">import { createBrowserRouter, useLoaderData, redirect } from 'react-router-dom';

// Loader: runs before the component renders
async function userLoader({ params }) {
  const user = await fetchUser(params.userId);
  if (!user) throw new Response('Not Found', { status: 404 });
  return user; // returned value is available via useLoaderData()
}

function UserProfile() {
  const user = useLoaderData(); // data from loader — no loading state needed!
  return &lt;div&gt;{user.name}&lt;/div&gt;;
}

const router = createBrowserRouter([
  { path: '/users/:userId', element: &lt;UserProfile /&gt;, loader: userLoader },

When a user clicks a <code>&lt;Link&gt;</code>, React Router calls <code>event.preventDefault()</code> to stop the browser's default navigation, then calls <code>history.pushState(null, '', path)</code> to update the URL bar without reloading. For browser back/forward navigation, the <code>popstate</code> event fires, the history library detects it, and the router updates accordingly. The key insight: all navigation is just state management — the URL is state, and route components are derived from that state.</p>

React Router wraps this API in its <code>history</code> library to provide a consistent interface for BrowserRouter, HashRouter, and MemoryRouter. When you call <code>navigate('/path')</code>, React Router calls <code>history.pushState</code>. When you call <code>navigate('/path', { replace: true })</code>, it calls <code>history.replaceState</code>. <code>BrowserRouter</code> uses <code>pushState</code>/<code>replaceState</code>; <code>HashRouter</code> uses <code>location.hash</code> changes; <code>MemoryRouter</code> maintains an in-memory array (for testing or non-browser environments like React Native). The router subscribes to history changes and updates its internal state, causing re-renders.</p>
<pre class="code-block">// What React Router does internally (simplified)
function createBrowserHistory() {
  let listeners = [];
  const history = {
    push(to) {
      window.history.pushState({ key: createKey() }, '', to);
      notifyListeners();
    },
    replace(to) {
      window.history.replaceState({ key: createKey() }, '', to);
      notifyListeners();
    },
    listen(fn) { listeners.push(fn); return () =&gt; { listeners = listeners.filter(l =&gt; l !== fn); }; },
  };
  window.addEventListener('popstate', notifyListeners); // back/forward buttons
  return history;
}</pre>

To display a preview, create an object URL with <code>URL.createObjectURL(file)</code> and use it as an <code>&lt;img src&gt;</code>. To upload, create a <code>FormData</code> object, append the file, and send via fetch/axios with <code>Content-Type: multipart/form-data</code> (or let the browser set this automatically). <pre class="code-block">function FileUpload() {
  const [file, setFile] = React.useState(null);
  const [preview, setPreview] = React.useState(null);
  const [uploading, setUploading] = React.useState(false);

  const handleFileChange = (e) =&gt; {
    const selectedFile = e.target.files[0];
    if (!selectedFile) return;
    setFile(selectedFile);
    setPreview(URL.createObjectURL(selectedFile)); // create preview URL
  };

  React.useEffect(() =&gt; {
    return () =&gt; { if (preview) URL.revokeObjectURL(preview); }; // cleanup
  }, [preview]);

  const handleUpload = async () =&gt; {
    if (!file) return;
    setUploading(true);
    const formData = new FormData();
    formData.append('file', file);
    await fetch('/api/upload', { method: 'POST', body: formData });
    setUploading(false);
  };

  return (
    &lt;div&gt;
      &lt;input type="file" accept="image/*" onChange={handleFileChange} /&gt;
      {preview &amp;&amp; &lt;img src={preview} alt="preview" style={{ width: 200 }} /&gt;}
      &lt;button onClick={handleUpload} disabled={!file || uploading}&gt;Upload&lt;/button&gt;
    &lt;/div&gt;
  );
}</pre>

          <p>WYSIWYG (What You See Is What You Get) editors are complex components that render formatted text in a <code>contentEditable</code> div. In React, you integrate third-party editor libraries since building one from scratch is extremely complex. Popular options: TipTap (headless, based on ProseMirror, excellent React integration with hooks API), Quill (mature, widely used, requires a wrapper component), Slate.js (highly customizable, React-native), Draft.js (Facebook, unmaintained), and TinyMCE/CKEditor (full-featured commercial options). The integration challenge: these editors are controlled differently from standard inputs; use <code>Controller</code> from React Hook Form to wrap them. The editor's value is typically HTML or JSON; handle this by converting between formats for storage and display.</p>
<pre class="code-block">// TipTap integration with React Hook Form
import { useEditor, EditorContent } from '@tiptap/react';
import StarterKit from '@tiptap/starter-kit';
import { Controller } from 'react-hook-form';

function RichTextEditor({ name, control, defaultValue = '' }) {
  return (
    &lt;Controller
      name={name}
      control={control}
      defaultValue={defaultValue}
      render={({ field }) =&gt; {
        const editor = useEditor({
          extensions: [StarterKit],
          content: field.value,
          onUpdate: ({ editor }) =&gt; field.onChange(editor.getHTML()),
        });
        return (
          &lt;div className="editor-wrapper"&gt;
            &lt;MenuBar editor={editor} /&gt;
            &lt;EditorContent editor={editor} /&gt;
          &lt;/div&gt;
        );
      }}
    /&gt;
  );
}</pre>
