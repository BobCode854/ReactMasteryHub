React Forms

Controlled forms, uncontrolled refs, React Hook Form, and validation patterns

Controlled Forms

In controlled forms, React state is the single source of truth for every input. Each change goes through setState, giving you full control for real-time validation and conditional UI.

Key idea: value={state} + onChange handler — React "owns" the input value. 
function LoginForm() {
  const [formData, setFormData] = useState({ email: '', password: '' });
  const [errors, setErrors] = useState({});
  const [loading, setLoading] = useState(false);

  const validate = () => {
    const errs = {};
    if (!formData.email.includes('@')) errs.email = 'Invalid email';
    if (formData.password.length < 8) errs.password = 'Min 8 characters';
    return errs;
  };

  const handleChange = (e) => {
    const { name, value } = e.target;
    setFormData(prev => ({ ...prev, [name]: value }));
    if (errors[name]) setErrors(prev => ({ ...prev, [name]: '' }));
  };

  const handleSubmit = async (e) => {
    e.preventDefault();
    const errs = validate();
    if (Object.keys(errs).length) { setErrors(errs); return; }
    setLoading(true);
    await loginAPI(formData);
    setLoading(false);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input name="email" value={formData.email} onChange={handleChange} />
      {errors.email && <span className="text-danger">{errors.email}</span>}
      <input name="password" type="password" value={formData.password} onChange={handleChange} />
      {errors.password && <span className="text-danger">{errors.password}</span>}
      <button type="submit" disabled={loading}>
        {loading ? 'Logging in...' : 'Login'}
      </button>
    </form>
  );
}

Uncontrolled Forms with useRef

Uncontrolled forms let the DOM manage input values. You read them on submit via refs — useful for simple forms, file inputs, or integrating with non-React libraries.

When to use: Quick forms with no real-time validation, file uploads, or third-party DOM integrations. 
function QuickForm() {
  const nameRef = useRef(null);
  const emailRef = useRef(null);

  const handleSubmit = (e) => {
    e.preventDefault();
    const data = {
      name: nameRef.current.value,
      email: emailRef.current.value,
    };
    console.log(data);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input ref={nameRef} defaultValue="" />
      <input ref={emailRef} type="email" />
      <button type="submit">Submit</button>
    </form>
  );
}

React Hook Form

React Hook Form minimises re-renders by using uncontrolled inputs under the hood while exposing a clean, declarative validation API. It's the performance-first choice for complex forms.

Performance: RHF only re-renders the field that changed — not the entire form — making it 10-20x faster than fully controlled forms for large forms. 
import { useForm } from 'react-hook-form';

function RegistrationForm() {
  const {
    register,
    handleSubmit,
    formState: { errors, isSubmitting },
    watch,
    reset
  } = useForm();

  const password = watch('password');

  const onSubmit = async (data) => {
    await registerUser(data);
    reset();
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input
        {...register('email', {
          required: 'Email required',
          pattern: { value: /^\S+@\S+$/i, message: 'Invalid email' }
        })}
      />
      {errors.email && <span>{errors.email.message}</span>}

      <input
        type="password"
        {...register('password', {
          required: true,
          minLength: { value: 8, message: 'Min 8 chars' }
        })}
      />

      <input
        type="password"
        {...register('confirmPassword', {
          validate: value => value === password || 'Passwords must match'
        })}
      />

      <button type="submit" disabled={isSubmitting}>
        {isSubmitting ? 'Registering...' : 'Register'}
      </button>
    </form>
  );
}

Tips & Common Mistakes

Always call e.preventDefault()
Without it, the browser reloads the page on form submit — the most common beginner mistake.
Use name attribute on inputs
In controlled forms, name lets a single handler manage all fields: setFormData(prev => ({...prev, [e.target.name]: e.target.value})).
Validate on blur for better UX
Show errors when the user leaves a field (onBlur), not while they're still typing.
Don't set value without onChange
A controlled input with value but no onChange becomes read-only and throws a React warning.
Avoid mixing controlled & uncontrolled
Switching an input from controlled to uncontrolled (or vice versa) at runtime causes React warnings and unpredictable behaviour.
Don't forget to reset after submit
Failing to clear form state after a successful submission leaves stale data in the form.

Interview Questions

What is the difference between a controlled and an uncontrolled component?
A controlled component has its value managed by React state — every change flows through setState. An uncontrolled component stores its own value in the DOM; you read it via a ref when needed (e.g., on submit).
Why does React Hook Form outperform Formik for large forms?
RHF uses uncontrolled inputs and only triggers a re-render for the specific field that changed. Formik subscribes all fields to a shared Formik state object, causing the entire form to re-render on every keystroke.
How do you handle cross-field validation (e.g., confirm password)?
With RHF, use watch('password') to read the password field value, then pass a validate function to the confirm-password field that compares the two values. With Yup, use .oneOf([Yup.ref('password')]).
When would you choose uncontrolled inputs over controlled?
Uncontrolled inputs are preferred for: file inputs (the browser controls FileList), simple one-off forms with no real-time validation, and integrating with third-party DOM libraries that manipulate the input directly.
How do you prevent a form from submitting when validation fails?
Call e.preventDefault() to suppress the native submit, then run your validation. If errors exist, update error state and return early. With RHF, handleSubmit automatically prevents submission if any registered validation rule fails.
How do you integrate Zod or Yup schema validation with React Hook Form?
Install the corresponding resolver (@hookform/resolvers/zod or /yup) and pass it as the resolver option to useForm: useForm({ resolver: zodResolver(schema) }). RHF will run the schema against the full form object on submit and on each field change.
