Section 21 Mobile Development

React Native

Build native iOS & Android apps using React concepts you already know — same mental model, different render target.

What is React Native?

React Native lets you build truly native mobile apps using React. Instead of rendering HTML elements in a browser, it renders native platform UI components — UIKit on iOS and Android Views on Android.

Same React, Different Output
  • React Web → renders HTML/CSS/JS → painted by browser engine
  • React Native → renders native iOS / Android components → painted by OS directly
  • Same component model, hooks, props, state, context, Redux — all carry over
  • JavaScript runs on a separate thread (JS Bridge or JSI in new architecture)
React Web
  • Renders HTML elements
  • Styled with CSS files
  • Runs in browser
  • React Router for navigation
  • onClick events
  • Any layout system (Grid, Flexbox)
React Native
  • Renders native components
  • Styled with StyleSheet API
  • Runs on iOS & Android
  • React Navigation library
  • onPress events
  • Flexbox only (no Grid)

Key Differences: React vs React Native

Concept React (Web) React Native
Layout CSS Flexbox / Grid StyleSheet (Flexbox only, no Grid)
Text <p>, <span>, <h1> <Text> — all text MUST use Text component
Input <input> <TextInput>
Scroll overflow: scroll CSS <ScrollView> or <FlatList>
Images <img src="..."> <Image source={require('./img.png')}/>
Touch onClick onPress (TouchableOpacity)
Navigation React Router React Navigation
State Mgmt Any — Redux, Zustand, Context Same — any state library works
Styling CSS files / CSS-in-JS StyleSheet.create({})
Animations CSS / Framer Motion Animated API / Reanimated
Div equivalent <div> <View>
Button <button> <Button> or <TouchableOpacity>

Core Components

React Native provides a set of built-in components that map directly to native platform widgets. Here's a practical example using FlatList, TouchableOpacity, StyleSheet, and Text:

import { View, Text, StyleSheet, TouchableOpacity, FlatList } from 'react-native'; function ItemList({ items, onPress }) { return ( <FlatList data={items} keyExtractor={item => item.id.toString()} renderItem={({ item }) => ( <TouchableOpacity style={styles.item} onPress={() => onPress(item)}> <Text style={styles.title}>{item.title}</Text> <Text style={styles.subtitle}>{item.subtitle}</Text> </TouchableOpacity> )} /> ); } const styles = StyleSheet.create({ item: { padding: 16, marginHorizontal: 16, marginVertical: 8, backgroundColor: '#fff', borderRadius: 12, shadowColor: '#000', shadowOffset: { width: 0, height: 2 }, shadowOpacity: 0.1, shadowRadius: 4, elevation: 3, // Android shadow }, title: { fontSize: 16, fontWeight: '700', color: '#1e293b' }, subtitle: { fontSize: 14, color: '#64748b', marginTop: 4 }, });
Essential Native Components
  • View — container (like <div>)
  • Text — all text must be inside Text
  • TextInput — controlled text input
  • Image — local and remote images
  • ScrollView — scrollable container
  • FlatList — virtualized performant lists
  • SectionList — grouped lists with headers
  • TouchableOpacity — pressable with fade
  • Pressable — modern press handler
  • Modal — overlay modal
  • ActivityIndicator — loading spinner
  • Switch — toggle switch
StyleSheet Notes
  • No units — numbers are density-independent pixels
  • No CSS cascade — each component styles itself
  • Flexbox is default layout (column direction by default!)
  • elevation for Android shadow
  • shadowColor/Offset/Opacity/Radius for iOS
  • Use Platform.OS for platform-specific values
  • StyleSheet.create() validates styles at dev time
Flexbox default direction is column in React Native, unlike CSS where it's row. Plan your layouts accordingly.

Navigation with React Navigation

React Native doesn't have a built-in router. React Navigation is the community standard. It supports Stack, Tab, Drawer, and nested navigators.

import { NavigationContainer } from '@react-navigation/native'; import { createNativeStackNavigator } from '@react-navigation/native-stack'; import { createBottomTabNavigator } from '@react-navigation/bottom-tabs'; const Stack = createNativeStackNavigator(); const Tab = createBottomTabNavigator(); function MainTabs() { return ( <Tab.Navigator> <Tab.Screen name="Home" component={HomeScreen} /> <Tab.Screen name="Profile" component={ProfileScreen} /> <Tab.Screen name="Settings" component={SettingsScreen} /> </Tab.Navigator> ); } function App() { return ( <NavigationContainer> <Stack.Navigator> <Stack.Screen name="Main" component={MainTabs} options={{ headerShown: false }} /> <Stack.Screen name="Detail" component={DetailScreen} /> </Stack.Navigator> </NavigationContainer> ); }
Navigation Types
Stack Navigator
Push/pop screens, like browser history. Back button / swipe gesture.
Tab Navigator
Bottom tabs (iOS) or top tabs (Android). Switch between top-level screens.
Drawer Navigator
Hamburger menu that slides in from the side.
Nested Navigators
Combine: Stack inside Tab, Drawer wrapping Stack, etc.

State Management in React Native

The good news: state management is identical to React web. Redux Toolkit, Zustand, Jotai, React Query — all work in React Native without any changes.

Redux Toolkit (same as web)
import { useSelector, useDispatch } from 'react-redux'; import { increment } from './counterSlice'; function Counter() { const count = useSelector(s => s.counter.value); const dispatch = useDispatch(); return ( <TouchableOpacity onPress={() => dispatch(increment())}> <Text>Count: {count}</Text> </TouchableOpacity> ); }
Persistent Storage (AsyncStorage)
import AsyncStorage from '@react-native-async-storage/async-storage'; // Store data await AsyncStorage.setItem('userToken', token); // Retrieve data const token = await AsyncStorage.getItem('userToken'); // Remove data await AsyncStorage.removeItem('userToken');
AsyncStorage is the React Native equivalent of localStorage — but it's async (returns Promises). Use redux-persist with AsyncStorage adapter for persisted Redux state.

Platform-Specific Code

Sometimes iOS and Android need different styles or behavior. React Native provides two approaches:

Platform API (inline)
import { Platform } from 'react-native'; const styles = StyleSheet.create({ container: { paddingTop: Platform.OS === 'ios' ? 50 : 20, ...Platform.select({ ios: { shadowColor: '#000', shadowOffset: { width: 0, height: 2 }, }, android: { elevation: 5 }, default: { /* other platforms */ } }), }, });
Platform-specific files
// Button.ios.tsx — used on iOS // Button.android.tsx — used on Android // Button.tsx — fallback // React Native auto-picks the right file import Button from './Button'; // ↑ loads Button.ios.tsx on iOS // loads Button.android.tsx on Android
Common Platform Checks
  • Platform.OS === 'ios'
  • Platform.OS === 'android'
  • Platform.Version — OS version number
  • Platform.isPad — iPad detection
  • Platform.isTV — Apple TV / Android TV

New Architecture (JSI & Fabric)

Old Architecture (Bridge)
  • Async JSON serialization bridge
  • JS thread & UI thread communicate via messages
  • Can cause dropped frames on heavy operations
  • Three threads: JS, Native, Shadow
New Architecture (JSI)
  • JSI — direct C++ interface to JS engine
  • Synchronous calls between JS and native
  • Fabric — new concurrent rendering system
  • TurboModules — lazy-loaded native modules
  • Better performance, smaller memory footprint
New Architecture is enabled by default in React Native 0.74+ and Expo SDK 51+.

Interview Questions

1. What is the difference between React and React Native? How do they share the same mental model?
React Native uses the same component model, hooks, and JSX as React. The difference is the render target — React outputs HTML DOM elements; React Native outputs native iOS/Android views. The bridge (or JSI in new arch) translates JS calls to native APIs.
2. Why must all text in React Native be inside a <Text> component?
Native platforms don't have a concept of "raw text nodes" like browsers do. Android and iOS require text to be in a specific native text widget. React Native enforces this by requiring the Text component, which maps to UILabel (iOS) and TextView (Android).
3. What is the difference between ScrollView and FlatList?
ScrollView renders all children at once — good for small lists or mixed content. FlatList is virtualized — it only renders items visible on screen plus a buffer, making it efficient for large datasets. Always use FlatList for lists with unknown or large lengths.
4. What is the difference between Expo Managed Workflow and bare React Native?
Expo Managed gives you a pre-configured environment with no native code to manage — ideal for most apps. Bare React Native gives full access to native code (Java/Kotlin, Obj-C/Swift) for custom native modules. You can "eject" from Expo to bare when needed.
5. How does navigation work in React Native? Compare with React Router.
React Native doesn't have a URL-based router. React Navigation manages a navigation stack in memory. It supports Stack (push/pop), Tab (bottom/top tabs), and Drawer navigators, which can be nested. Unlike React Router, navigation state is kept in JS memory, not the browser URL.
6. What is the React Native New Architecture and how does JSI improve performance?
The old bridge used async JSON serialization between JS and native threads, causing potential jank. JSI (JavaScript Interface) allows direct synchronous C++ function calls from JS to native, eliminating serialization overhead. Fabric is the new renderer built on JSI, enabling concurrent features and synchronous layout reads.
